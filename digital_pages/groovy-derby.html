Groovy Derby

Role Lead Developer || Individual
Tools p5JS, Arduino, Laser Cutting, Soldering
Visit the GitHub repo↗ for this project

The Challenge

The concept for my project was inspired by the co-op game It Takes Two, where two game characters set out on a journey that is filled with cooperative minigames and obstacles which they have to overcome. As such, I wanted to create a minigame where the game is only “playable” if two people are communicating and working together. Combining with themes from arcade games such as a limited number of lives and endless generative spites, I created Groovy Derby where two players have to input the correct order of key combinations in order to score points in the game. In addition, I created two controllers that mapped to keyboard inputs using Arduino.

The Approach

I started by focusing on the visual feel of the game, since I wanted the characters to move smoothly and feel animated rather than stiff. I broke each character’s movement into small pieces and designed the animation frame by frame, which helped the motions feel more natural even though the characters stayed in place. This approach made the interactions between the two players feel coordinated and expressive, and it relieved a lot of my initial worry about the game looking blocky or unfinished.

Next, I worked on the game logic and player interaction. I built player and creature classes that checked user input against the required button combinations in real time. If the input matched, the creature’s sequence would shrink; if it didn’t, the creature kept moving closer until a life was lost. This made the gameplay feel responsive and kept the rules simple and clear. I also reused tools like Canva, free sound effects, and custom fonts to quickly polish the game without spending time reinventing assets.

Finally, I extended the project to a physical setup by connecting an Arduino button box to p5.js using serial communication. The Arduino handled all button inputs and sent signals directly to p5, where I mapped them to game actions and state changes. I had to carefully manage button on/off states to avoid accidental double inputs, but once that was resolved, the physical controls worked smoothly. Overall, given the short timeline, I’m happy with how the visuals, gameplay, and physical interaction came together, and I see clear paths for future improvements.

The Outcome

The device successfully detected and classified Parkinson’s-related movement patterns in real time using motion data analysis. It provided immediate visual feedback through onboard LEDs and an LCD screen, making the results easy to understand at a glance. This demonstrated that wearable sensor data can be used effectively to recognize and communicate different movement states in a simple and accessible way.
