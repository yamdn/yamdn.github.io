Spotify Genre Classification

Role Developer || Individual
Tools Pandas, Numpy, pyTorch, Jupiter Notebook
Visit the GitHub repo↗ for this project

The Challenge

Spotify provides an API that exposes detailed audio features for each song, including attributes like tempo, energy, and danceability. In this project, we used these features from a dataset of 50,000 randomly selected songs to train a model that predicts the genre of a song. By analyzing patterns in these audio characteristics, the system learns how different genres tend to sound and can classify new songs based on their musical features.

The Approach

First, I focused on preparing the dataset so the model could learn from meaningful information. I removed text-based fields like artist and track names since they don’t help a model identify musical style. I then handled missing and invalid values by predicting them from related audio features, rather than using simple averages, because song features are often skewed and interrelated. Categorical features were encoded, numeric values were scaled where appropriate, and the data was split into training and testing sets to ensure fair evaluation without data leakage.

Next, I explored the structure of the data using dimensionality reduction techniques to better understand how genres relate to one another. PCA and t-SNE were used to visualize patterns and feature importance, revealing that many genres overlap and share similar characteristics. These visualizations showed that features like popularity had strong influence, while others contributed less, and that higher-dimensional representations were needed to capture meaningful differences between genres.

Finally, I trained a Random Forest classifier to predict music genres, since it handles nonlinear relationships and high-dimensional data well. I tested multiple variations of the dataset, including standardized and reduced versions, and found that the model performed best using the original feature set without dimensionality reduction. While the model achieved a strong AUC score, the overlapping nature of genres limited accuracy, suggesting that additional data or improved feature extraction could further improve performance.

The Outcome

The project showed that Spotify’s audio features can be used to predict a song’s genre, even with overlapping characteristics across categories. Using a Random Forest classifier on the unstandardized dataset, the model achieved the highest AUC score of 0.9335, indicating strong overall predictive power. However, the accuracy and precision of 0.5658 revealed that genre overlap still made exact classification challenging. Despite this, the results demonstrate that the preprocessing and modeling approach was effective, and further improvements in feature extraction or additional training data could help increase accuracy and precision.
